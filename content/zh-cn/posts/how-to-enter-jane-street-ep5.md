如果你在 Jane Street 的网站上逛过，大概率会发现他们的 [puzzles 页面](https://www.janestreet.com/puzzles/)。每隔一两个月，他们会发一道新题。有时候是数论，有时候是组合博弈，有时候甚至连规则都模糊不清——而这恰恰是重点。

Jane Street 自己说得很清楚："解题这件事，虽然看起来抽象，但却是我们日常工作的核心。"在金融市场中识别新问题、想出（可能是全新的）方法来解决它们，这就是他们的工作。所以谜题既是招聘筛选器，也是文化信号。

本期拆解六大类在 Jane Street 面试和月度挑战中反复出现的谜题。每一类都有完整的解法、Python 模拟验证，以及最重要的——**能迁移回量化交易的可复用洞见**。

## 为什么 Jane Street 重视谜题

大多数量化公司考你心算和背公式。Jane Street 考的是不一样的：**面对陌生问题，你能不能清晰地思考？**

他们的谜题有几个共同 DNA：

- **小样例藏着大规律。** n=2 或 n=3 的情况总能手算。洞见通常从小样例中浮现。
- **信息是硬通货。** 很多谜题归根结底是：你*知道*什么 vs 你*假设*了什么？
- **总有反转。** 显而易见的答案通常是错的，或者至少不完整。
- **和真实数学相连。** "帽子游戏"本质上是纠错码。"有偏硬币"是 von Neumann 的把戏。这些不是玩具——它们和市场微结构、信号处理用的是同一套数学结构。

## 1. 100 个储物柜（计数与奇偶性）

**题目：** 100 个学生依次走过 100 个关闭的储物柜。第 1 个学生打开所有柜子。第 2 个切换每第 2 个柜子的状态。第 3 个切换每第 3 个……第 100 个切换第 100 个柜子。最终哪些柜子是打开的？

**解法：** 柜子 *k* 被切换的次数等于 *k* 的因子个数。大多数数的因子成对出现 (d, k/d)，所以切换偶数次，最终关闭。唯一例外：**完全平方数**——√k 和自己配对，导致因子个数为奇数。

打开的柜子：1, 4, 9, 16, 25, 36, 49, 64, 81, 100——恰好 10 个。

**深层洞见：** 这是一个*奇偶性论证*。你不需要模拟 100 个学生。你只需要问："什么时候因子数是奇数？"奇偶性论证在 Jane Street 谜题中无处不在——在交易中也一样（比如：某个价位的成交订单数是奇还是偶？这会影响仓位管理）。

## 2. n 个骰子的最大值期望（线性期望）

**题目：** 掷 n 个公平的六面骰子。最大值的期望是多少？

**解法：**

$$E[\max] = \sum_{k=1}^{6} k \cdot \left[\left(\frac{k}{6}\right)^n - \left(\frac{k-1}{6}\right)^n\right]$$

最大值等于 *k* 的概率 = 所有色子 ≤ k 的概率 − 所有色子 ≤ k−1 的概率。

| n | E[max] |
|---|--------|
| 1 | 3.500 |
| 2 | 4.472 |
| 3 | 4.958 |
| 10 | 5.824 |

**深层洞见：** "期望值"不等于"最可能值"。n=10 时期望最大值是 5.82，但*众数*是 6。Jane Street 会试探你是否混淆期望与众数、中位数。在交易中，你的盈亏分布很少对称——了解分布的*形状*和均值一样重要。

## 3. von Neumann 的公平硬币（信息论）

**题目：** 你有一枚有偏硬币，正面概率 *p* 未知（p ≠ 0.5）。如何生成一次公平的硬币投掷？

**解法（von Neumann, 1951）：** 掷两次：
- **正反** → 输出"正面"
- **反正** → 输出"反面"
- **正正或反反** → 丢弃，重来

原理：P(正反) = P(反正) = p(1−p)，与 *p* 无关。两种序列等概率，映射到正反即得完美公平的硬币。

每次公平输出期望投掷次数：1 / (2p(1−p))。p=0.9 时约 5.6 次。

**深层洞见：** 你可以从任何有偏源中提取公平随机性——即使偏置*未知*。这是**熵提取**的基础，直接关联信号处理、市场微结构（从嘈杂订单流中提取信号）和密码学。

## 4. 帽子游戏（合作博弈与编码理论）

这是 Jane Street 谜题的王冠。它和他们日常使用的数学直接相连。

**题目：** *n* 个人各戴一顶红或蓝帽子（均匀随机分配）。每个人能看到其他所有人的帽子，但看不到自己的。所有人同时选择猜自己的颜色或弃权。团队*获胜*的条件是至少一人猜对且无人猜错。如果任何人猜错或所有人都弃权，则*失败*。

最大获胜概率是多少？策略是什么？

**朴素策略（n=3）：** 指定一人永远猜"红"。获胜概率 50%。不太好。

**汉明码策略（n = 2^k − 1）：** 获胜概率 = n/(n+1)。

n=3 时 **75%**。n=7 时 **87.5%**。n=15 时 **93.75%**。

原理——和**纠错码**用的是同一套数学：

1. 把每种帽子分配看作二进制向量 **h** ∈ {0,1}ⁿ。
2. 构造汉明码的校验矩阵 **H**：第 *j* 列 = *j* 的二进制表示。
3. 玩家 *i* 计算观察到的"校验子"：**v**_i = 可见红帽对应列的 XOR。
4. 决策规则：
   - **v**_i = 0 → 猜"红"
   - **v**_i = 第 *i* 列 → 猜"蓝"
   - 其他 → **弃权**

**为什么有效：** 真实校验子 **s** = **H** · **h**。当 **s** ≠ 0 时，它恰好指向一个比特位置——恰好一个玩家触发猜测条件且猜对。当 **s** = 0 时，所有玩家都猜"红"且全部猜错。但 **s** = 0 的概率仅 1/(n+1)。

**深层洞见：** 这*就是*纠错码的工作原理。汉明码通过计算校验子来检测和纠正单比特错误。Jane Street 在交易系统中使用类似的冗余和错误检测逻辑——确保单个数据错误不会级联成一笔错误交易。谜题不是玩具；它是同一个数学结构，只是把帽子换成了数据包。

## 5. 吸收随机游走（马尔可夫链）

**题目：** 一个醉汉从数轴 {0, 1, ..., N} 上的位置 *k* 出发。每步等概率向左或向右走。到达 0 或 N 时停止（吸收壁）。期望步数是多少？

**解法：** E[从位置 *k* 出发的步数] = *k*(N − *k*)。

N=10, k=5：E = 25 步。N=100, k=50：E = 2500 步。

**深层洞见：** 这就是**赌徒破产**问题——有限资本且没有优势的交易者*必然*最终破产的数学原因。Jane Street 的整个商业模式建立在拥有正期望优势上；没有优势，随机游走的数学毫不留情。公式 k(N−k) 还告诉你期望持续时间与屏障间距呈二次关系——这就是为什么仓位管理如此重要：仓位相对资本越大，"屏障"越近。

## 6. 收集者问题（调和级数）

**题目：** 每个麦片盒里有一张 *n* 种不同优惠券中的一张（均匀随机）。期望买多少盒才能集齐全部 *n* 种？

**解法：** E[盒数] = *n* · H_n = *n* · Σ(1/k, k=1..n)。

| n | E[盒数] |
|---|---------|
| 10 | 29.3 |
| 26 | 100.2 |
| 52 | 236.0 |

**深层洞见：** 最后几张优惠券指数级昂贵。从差 1 张到集齐，平均需要 *n* 盒。这种"最后一英里"问题无处不在：做市中大额订单的最后几张合约最贵；组合优化中增加更多资产的边际收益递减；采样中覆盖稀有事件代价极高。

## 六步方法论

做完几十道 Jane Street 谜题后，我总结了一套可复用的解题流程：

1. **仔细读题。** JS 谜题在措辞中总藏着一个反转。*真正*问的是什么？
2. **先做小样例。** 手算 n=2, n=3。规律通常很快浮现。
3. **信息边界。** 问自己：有多少比特的信息可用？理论最优结果是什么？这会约束你的搜索空间。
4. **对称性。** 寻找不变量、对称结构或奇偶性论证。它们经常把难题变成平凡题。
5. **模拟验证。** 写一个快速的 Python 模拟。如果模拟和公式不一致，其中一定有错。
6. **证明后推广。** 得到答案后，问："如果改变 X 会怎样？" JS 面试喜欢追问来测试你的理解深度。

## 从谜题到交易

谜题和 Jane Street 实际工作之间的联系不是比喻性的——是字面意义上的：

- **线性期望** → 给衍生品定价、计算投资组合希腊字母
- **奇偶性论证** → 仓位检查、订单验证
- **纠错码** → 确保行情数据完整性
- **随机游走** → 建模价格路径、风险管理
- **调和级数** → 理解成交率和市场冲击
- **信息论** → 从嘈杂订单流中提取信号

当 Jane Street 说"解题感觉很好"时，他们不是在卖萌。谜题是他们每天面对的量化问题的精华——去掉了金融术语，还原为数学本质。
