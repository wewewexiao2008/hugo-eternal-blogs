上一期我梳理了 Jane Street 是什么样的公司、他们找什么样的人。他们测试的最重要的技术领域是什么？**概率论**。不是金融，不是机器学习，甚至不是算法——是概率。

这期是一个训练营，覆盖在量化面试中反复出现的核心概念。每个结论我都用蒙特卡洛模拟验证过，因为那是我检验自己推理的方式。

## 为什么概率最重要？

Jane Street 的面试页面写着：*"Finance background: optional."* 他们真正测试的是你如何在不确定性下思考。市场充满噪声，价格是随机的，每一笔交易决策都是一次概率计算。

做市的数学基础说起来很简单：观察一个带噪声的信号，更新你对真实价值的估计，然后决定下多少注。这就是条件期望。其他一切都在此之上构建。

## 你需要的五个工具

### 工具一：条件概率与贝叶斯定理

**公式：** P(A|B) = P(A∩B) / P(B)

**经典问题：** 一种疾病影响千分之一的人。检测准确率 99%（敏感性和特异性都是）。你检测为阳性。你真正得病的概率是多少？

大多数人回答约 99%。实际答案是约 **9%**。

```
P(D|+) = P(+|D) · P(D) / P(+)
P(+)   = 0.99 × 0.001 + 0.01 × 0.999 = 0.01098
P(D|+) = 0.00099 / 0.01098 ≈ 9.02%
```

先验概率主导了结果。即使检测很优秀，罕见疾病意味着大多数阳性是假阳性。这就是**基础概率谬误**（base rate fallacy），它在交易中有直接对应：如果事件本身极其罕见，一个很强的信号也不代表太多。

我用 20 万次试验模拟了不同先验下的结果：

| 先验 P(D) | P(得病 | 阳性) |
|-----------|-------------------|
| 0.001     | 9.02%             |
| 0.01      | 50.0%             |
| 0.05      | 83.9%             |
| 0.10      | 91.7%             |

同一个 99% 准确率的检测，先验不同，结果天差地别。在交易中，你的"先验"就是你对公允价值的现有模型。单次价格跳动不该推翻一个校准好的先验——但应该按比例更新它。

**蒙提霍尔问题**（三门问题）是另一个著名的条件概率题。你选一扇门，主持人打开另一扇后面是羊的门，你应该换门——胜率从 1/3 翻倍到 2/3。模拟验证：20 万次试验中，换门胜率 0.6666，不换 0.3334。关键洞察是主持人的行为包含了*信息*——他不能打开有车的那扇门。根据这个信息更新概率，结果就变了。

### 工具二：期望的线性性

**公式：** E[aX + bY] = a·E[X] + b·E[Y]，永远成立。即使 X 和 Y 不独立。

这是概率面试中最强大的工具，但它太简单了，容易被低估。

**集卡问题（Coupon Collector）：** 掷一个公平的 6 面骰子，平均要掷多少次才能看到所有 6 个面？

设 T = 总掷数。拆成 T₁ + T₂ + ... + T₆，其中 Tᵢ = 已经看到 (i-1) 个面之后，看到第 i 个新面需要的掷数。

每个 Tᵢ 服从几何分布，参数 p = (7-i)/6：

```
E[T] = 6/6 + 6/5 + 6/4 + 6/3 + 6/2 + 6/1
     = 6 × (1 + 1/2 + 1/3 + 1/4 + 1/5 + 1/6)
     = 6 × 2.45 = 14.70
```

10 万次模拟：**14.71**。吻合。

**为什么线性性重要：** Tᵢ 之间显然不独立——你不可能在看到第 2 个新面之前看到第 3 个。但线性性不在乎独立性。E[T₁+T₂+...+T₆] = E[T₁] + E[T₂] + ... + E[T₆]，无需建模联合分布。

**另一个应用：** 两个骰子 X 和 Y，E[max(X,Y)] 是多少？

```
E[max(X,Y)] = Σ k · P(max = k)
P(max = k) = (k/6)² - ((k-1)/6)²
E[max] = 161/36 ≈ 4.472
```

模拟：10 万次试验得到 **4.473**。

### 工具三：模式等待时间（Penney's Game）

这道题在面试中反复出现。

**问题：** 抛公平硬币，平均需要抛多少次才能看到模式 HT？那 HH 呢？

| 模式 | 期望等待次数 |
|------|-------------|
| HT   | 4           |
| HH   | 6           |
| HTH  | 10          |
| HHT  | 8           |
| HTHH | 18          |

差异的根源是**自重叠（self-overlap）**。如果你在等 HH，收到了 H 然后是 T，你就回到零了——没有任何进展。但如果你在等 HT，收到 H 然后是 H，第二个 H 可能是下一个 HT 的开头——你部分地保留了进展。

通用公式使用 **Conway 算法**（以 John Horton Conway 命名）：

```
E[等待模式 A] = Σ 2^k，其中 k 遍历 overlap(A, A)
```

"重叠"衡量模式 A 的前缀和后缀在哪里匹配。HT 没有自重叠 → 2² = 4。HH 在位置 1 重叠 → 2² + 2¹ = 6。HTH 在位置 0 和 2 重叠 → 2³ + 2¹ = 10。

我的 20 万次模拟确认：HT = 4.00, HH = 5.98, HTH = 10.00, HHT = 8.00。

**Penney's Game：** 玩家 A 选一个 3 次抛掷模式，玩家 B 选另一个。不断抛硬币直到某个模式出现。玩家 B 总能利用重叠结构选出一个胜率 > 1/2 的模式。这种非传递性是很好的面试话题。

### 工具四：赌徒破产与可选停时定理

你有 $i，对手有 $(N-i)。每轮你以概率 p 赢 $1 或以概率 (1-p) 输 $1。你在赢光之前破产的概率是多少？

**公平赌局（p = 0.5）：** P(达到 N) = i/N

**有偏赌局（p ≠ 0.5）：** P(达到 N) = (1-(q/p)^i) / (1-(q/p)^N)，其中 q = 1-p

| 起始资金 $i | 总额 $N | p    | P(全赢) |
|------------|---------|------|---------|
| 50         | 100     | 0.50 | 0.5000  |
| 50         | 100     | 0.49 | 0.1197  |
| 50         | 100     | 0.48 | 0.0277  |

1% 的劣势（p = 0.49 而非 0.50）把胜率从 50% 降到 12%。2% 的劣势降到不到 3%。**微小的优势在大量试验中会产生巨大的累积效应。**

这对交易有深刻启示：如果你有哪怕很微小的负期望，长期来看你必然会破产。Jane Street 的整个商业模式建立在拥有*正*期望——无论多么小——并参与极大量交易的基础上。

**可选停时定理**告诉我们：对于公平赌局，没有任何停时策略能给你正的期望利润。你不可能通过择时击败一个公平赌局。这直接否定了"输了就翻倍"的马丁格尔策略——资金有限时，期望利润恰好是 $0。我用 $1,023 起始资金、1 万次试验验证了这一点：胜率 99.9%，但平均利润 $0.00。

Jane Street 的优势来自**信息**（更好的条件期望估计）和**速度**（更快地计算这些期望），而不是巧妙的下注时机。

### 工具五：条件期望——交易的核心

这是做市的数学心脏。

**设定：** X ~ Uniform(0,1) 代表资产的"真实价值"。你观察到 Y = X + noise，其中 noise ~ Uniform(-0.5, 0.5)。给定 Y = y，E[X|Y=y] 是多少？

这就是做市商做的事：观察带噪声的价格信号，估计真实价值，在估计值两侧设定买卖价。

我用拒绝采样数值计算了这个结果：

| 观察到的 Y | E[X \| Y] |
|-----------|-----------|
| 0.00      | 0.250     |
| 0.25      | 0.375     |
| 0.50      | 0.500     |
| 0.75      | 0.625     |
| 1.00      | 0.750     |

在 Y = 0.5（中间位置），估计值恰好是 0.5——噪声相互抵消。但在极端值处，估计值向中心收缩，因为先验（X ~ Uniform(0,1)）约束了真实价值的范围。这就是贝叶斯更新的实际运作。

**与交易的联系：** Jane Street 观察订单流、价格变动和市场数据（即"Y"）。他们的模型比竞争对手更快更准地计算出 E[公允价值 | 观察值]。买卖价差补偿了剩余的不确定性。这就是他们的生意。

## 费米问题加餐

量化面试喜欢估算题。经典一例："芝加哥有多少钢琴调音师？"

方法是乘一串估计值，每一步都有很大的误差范围。误差往往会相互抵消，最终结果通常落在真实值的一个数量级以内。

这与交易中的**敏感性分析**直接相关：模型中哪个因子最重要？某个输入差 2 倍，输出变 2 倍还是 200 倍？理解哪个参数主导，比把任何一个数字算准更有价值。

## 我的练习方法

1. **先解析求解。** 写出数学过程，说明假设。
2. **模拟验证。** 如果模拟和数学不一致，其中一个错了——通常是数学。
3. **改变参数。** p = 0.49 和 p = 0.50 有什么区别？10 枚硬币和 100 枚呢？
4. **大声说出来。** 面试是协作式的。出声思考不是可选项——它就是重点所在。
